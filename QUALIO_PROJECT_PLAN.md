## Final Product

Qualio is a single-page web application designed for wine buyers at restaurants, hotels, and online retailers. The product removes manual cross-referencing from the sourcing workflow. A buyer photographs a wine label — that is the only input required. Everything else is computed automatically.

The interface is intentionally minimal. A dark, editorial design with a single upload zone and a clear call to action. Once the label is uploaded and the analysis runs, the page reveals four sections in sequence.

**What wine is it** shows the CNN output: the predicted wine style and country of origin with a confidence score, and a probability breakdown across all five countries the model was trained on. The buyer sees immediately whether the model is confident or uncertain, and across which categories it is splitting its decision.

**How people rate it** shows the BiLSTM output: the predicted quality tier — Entry, Premium, or Exceptional — alongside the full star rating distribution pulled from WineSensed across all Vivino reviews for that vintage. Rather than showing a single number, the buyer sees the shape of opinion. A wine with 58% five-star reviews and almost no negative reviews tells a different story than one with evenly spread ratings, even if both average 4.1.

**Wine fact card** shows the structured data retrieved from WineSensed for the matched vintage: the wine name, grape composition, region, vintage year, average Vivino rating with total review count, price in USD, and alcohol percentage. This is not model output — it is direct retrieval. The two model predictions (country and quality tier) are used to frame the card with a combined badge, for example "Exceptional · French Red."

**Buyer recommendation** is a plain-language synthesis generated by the business integration layer. It combines the CNN country prediction and the BiLSTM quality tier into an actionable stocking decision — whether to prioritise acquisition, treat as standard stock, or flag as a bulk candidate — with a note on portfolio positioning.

**Wines that taste like this one** is the section that distinguishes Qualio from any standard ML classifier. Three cards appear below the fact card, each showing a wine that human participants placed closest to the analysed bottle during the WineSensed napping experiment. The similarity percentage is derived from Euclidean distance in the 2D taste coordinate space from `napping.csv`, not from any algorithmic recommendation engine. A taste space map visualises the full napping coordinate plane, with the analysed wine highlighted in gold and dashed lines connecting it to its three nearest neighbours. For vintages outside the 108-wine napping subset, the system falls back to grape family and price tier proximity.

The product runs as a FastAPI application serving the HTML frontend as a static file. The CNN runs live at inference time on the uploaded image. The BiLSTM does not run at inference time — it runs in batch during preprocessing and its predictions are stored in an enriched CSV that the API queries on demand. This separation means the API response is fast even without a GPU on the deployment server.

## 1. Project Overview

### What this project does

Qualio is a wine sourcing intelligence tool. A buyer uploads a label photo. The system returns:

- **What wine it is** — CNN classifies wine style and country of origin from the label image
- **How people rate it** — BiLSTM reads Vivino user reviews from WineSensed and classifies quality tier
- **Wine fact card** — retrieved from WineSensed: name, grape, region, price, alcohol, Vivino rating
- **Similar wines** — computed from napping.csv pairwise taste distances (human-derived)

### Architecture in one line

```
Label image → CNN → wine style + country
     ↓
Vintage ID → lookup WineSensed → wine attributes + reviews
                                        ↓
                               BiLSTM → quality tier (Entry / Premium / Exceptional)
                                        ↓
                     Napping coordinates → similar wines by taste distance
                                        ↓
                              Combined sourcing brief
```

### What the rubric requires

| Requirement | Our implementation |
|---|---|
| CNN for image classification | ResNet-18, wine style + country, 5 classes |
| RNN for text classification | BiLSTM, quality tier from reviews, 3 classes |
| Same business domain | Both use WineSensed wine data |
| Business integration layer | Sourcing brief combining both outputs |
| Interactive prototype | FastAPI + Qualio HTML frontend |
| Two CNN approaches | Scratch CNN + ResNet-18 transfer learning |
| Two RNN variations | Unidirectional LSTM + BiLSTM |
| Explainability | Grad-CAM (CNN) + attention weights (LSTM) |

#### Text preprocessing

```python
import numpy as np
from collections import Counter
import nltk
nltk.download('punkt')
nltk.download('stopwords')

MAX_LEN  = 200    # pad/truncate to this length
VOCAB_SIZE = 20000

def build_vocab(texts, max_vocab=VOCAB_SIZE):
    counter = Counter()
    for text in texts:
        tokens = nltk.word_tokenize(text.lower())
        counter.update(tokens)
    vocab = {'<PAD>': 0, '<UNK>': 1}
    for word, _ in counter.most_common(max_vocab - 2):
        vocab[word] = len(vocab)
    return vocab

def encode(text, vocab, max_len=MAX_LEN):
    tokens = nltk.word_tokenize(text.lower())
    ids = [vocab.get(t, 1) for t in tokens]  # 1 = <UNK>
    # Pad or truncate
    if len(ids) < max_len:
        ids += [0] * (max_len - len(ids))
    else:
        ids = ids[:max_len]
    return ids

# Report truncation
pct_truncated = (df['review_len'] > MAX_LEN).mean() * 100
print(f"Reviews truncated at {MAX_LEN} tokens: {pct_truncated:.1f}%")
```

#### GloVe embeddings

```python
def load_glove(glove_path, vocab, embed_dim=100):
    matrix = np.zeros((len(vocab), embed_dim))
    found = 0
    with open(glove_path, encoding='utf-8') as f:
        for line in f:
            parts = line.split()
            word = parts[0]
            if word in vocab:
                matrix[vocab[word]] = np.array(parts[1:], dtype=np.float32)
                found += 1
    print(f"GloVe coverage: {found}/{len(vocab)} words ({found/len(vocab)*100:.1f}%)")
    return matrix
```

#### Approach 1 — Scratch CNN (3 convolutional blocks)

```python
import torch
import torch.nn as nn

class ScratchCNN(nn.Module):
    def __init__(self, num_classes=5):
        super().__init__()

        self.features = nn.Sequential(
            # Block 1
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),           # 224 → 112

            # Block 2
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),           # 112 → 56

            # Block 3
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),           # 56 → 28
        )

        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool2d((1, 1)),  # global average pooling
            nn.Flatten(),
            nn.Dropout(0.4),
            nn.Linear(128, num_classes)
        )

    def forward(self, x):
        return self.classifier(self.features(x))

model = ScratchCNN(num_classes=5)
print(f"Trainable params: {sum(p.numel() for p in model.parameters() if p.requires_grad):,}")
```

#### Approach 2 — ResNet-18 transfer learning

```python
import torchvision.models as models

def build_resnet18(num_classes=5, freeze_backbone=True):
    model = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1)

    if freeze_backbone:
        for param in model.parameters():
            param.requires_grad = False

    # Replace classification head
    in_features = model.fc.in_features
    model.fc = nn.Sequential(
        nn.Dropout(0.3),
        nn.Linear(in_features, num_classes)
    )

    return model

# Phase 1: frozen backbone — train only the head
model_resnet = build_resnet18(freeze_backbone=True)

# Phase 2 (optional): unfreeze last 2 conv blocks for fine-tuning
def unfreeze_last_blocks(model, n_blocks=2):
    layers = [model.layer4, model.layer3][:n_blocks]
    for layer in layers:
        for param in layer.parameters():
            param.requires_grad = True
```

#### Training loop

```python
from torch.optim import Adam
from torch.optim.lr_scheduler import ReduceLROnPlateau

def train_model(model, train_loader, val_loader, epochs=20, lr=1e-3):
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model = model.to(device)

    criterion = nn.CrossEntropyLoss()
    optimizer = Adam(filter(lambda p: p.requires_grad, model.parameters()), lr=lr)
    scheduler = ReduceLROnPlateau(optimizer, patience=3, factor=0.5)

    history = {'train_loss': [], 'val_loss': [], 'train_acc': [], 'val_acc': []}
    best_val_loss = float('inf')

    for epoch in range(epochs):
        model.train()
        train_loss, train_correct = 0, 0

        for images, labels in train_loader:
            images, labels = images.to(device), labels.to(device)
            optimizer.zero_grad()
            outputs = model(images)
            loss = criterion(outputs, labels)
            loss.backward()
            optimizer.step()
            train_loss += loss.item()
            train_correct += (outputs.argmax(1) == labels).sum().item()

        # Validation
        model.eval()
        val_loss, val_correct = 0, 0
        with torch.no_grad():
            for images, labels in val_loader:
                images, labels = images.to(device), labels.to(device)
                outputs = model(images)
                val_loss += criterion(outputs, labels).item()
                val_correct += (outputs.argmax(1) == labels).sum().item()

        # Log
        t_loss = train_loss / len(train_loader)
        v_loss = val_loss / len(val_loader)
        t_acc  = train_correct / len(train_loader.dataset)
        v_acc  = val_correct / len(val_loader.dataset)

        history['train_loss'].append(t_loss)
        history['val_loss'].append(v_loss)
        history['train_acc'].append(t_acc)
        history['val_acc'].append(v_acc)

        scheduler.step(v_loss)
        print(f"Epoch {epoch+1:02d} | train loss {t_loss:.4f} acc {t_acc:.3f} | val loss {v_loss:.4f} acc {v_acc:.3f}")

        # Save best model
        if v_loss < best_val_loss:
            best_val_loss = v_loss
            torch.save(model.state_dict(), 'weights/cnn_resnet18_best.pt')

    return history

# Save final weights
torch.save(model_resnet.state_dict(), 'weights/cnn_resnet18.pt')
```

#### Approach 1 — Unidirectional LSTM

```python
class WineLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim=100, hidden_dim=256,
                 num_layers=1, num_classes=3, dropout=0.3,
                 pretrained_embeddings=None, bidirectional=False):
        super().__init__()

        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        if pretrained_embeddings is not None:
            self.embedding.weight.data.copy_(
                torch.tensor(pretrained_embeddings, dtype=torch.float)
            )

        self.lstm = nn.LSTM(
            input_size=embed_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0,
            bidirectional=bidirectional
        )

        lstm_out_dim = hidden_dim * (2 if bidirectional else 1)

        self.classifier = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(lstm_out_dim, num_classes)
        )

    def forward(self, x):
        embedded = self.embedding(x)
        output, (hidden, _) = self.lstm(embedded)

        if self.lstm.bidirectional:
            # Concatenate final forward and backward hidden states
            hidden = torch.cat([hidden[-2], hidden[-1]], dim=1)
        else:
            hidden = hidden[-1]

        return self.classifier(hidden)
```

#### RNN runs in batch — not at inference time

```python
# After training, run BiLSTM on all reviews in the dataset
# Save predicted tier back to the dataframe
# This enriched CSV is what the FastAPI backend uses at runtime

def batch_predict_tiers(model, df, vocab, device):
    model.eval()
    tiers = []
    confidences = []

    with torch.no_grad():
        for review in df['review'].fillna(''):
            ids = torch.tensor([encode(review, vocab)], dtype=torch.long).to(device)
            logits = model(ids)
            probs = torch.softmax(logits, dim=1)
            tier = probs.argmax(1).item()
            conf = probs.max(1).values.item()
            tiers.append(tier)
            confidences.append(conf)

    df['predicted_tier'] = tiers
    df['tier_confidence'] = confidences
    return df

enriched_df = batch_predict_tiers(bilstm_model, df, vocab, device)
enriched_df.to_csv('data/enriched_dataset.csv', index=False)
```

### Step 6 — Business integration

**Notebook:** `05_integration.ipynb`
**Time estimate:** 3 hours

#### Napping similarity

```python
import pandas as pd
import numpy as np
from scipy.spatial.distance import euclidean

napping = pd.read_csv('data/napping.csv')

def get_similar_wines(vintage_id, enriched_df, napping_df, top_n=3):
    """
    Find wines that tasted most similar in the human napping experiment.
    Falls back to grape/region proximity if wine not in napping subset.
    """
    # Get this wine's experiment_id
    wine_row = enriched_df[enriched_df['vintage_id'] == vintage_id]
    if wine_row.empty:
        return []

    exp_id = wine_row['experiment_id'].values[0]
    if pd.isna(exp_id):
        return _fallback_similar(wine_row, enriched_df, top_n)

    # Get this wine's napping coordinates (average across participants)
    target = napping_df[napping_df['experiment_id'] == exp_id][['coor1','coor2']].mean()
    if target.isna().any():
        return _fallback_similar(wine_row, enriched_df, top_n)

    # Compute Euclidean distances to all other wines in napping
    distances = []
    for exp_id_other in napping_df['experiment_id'].unique():
        if exp_id_other == exp_id:
            continue
        other = napping_df[napping_df['experiment_id'] == exp_id_other][['coor1','coor2']].mean()
        if other.isna().any():
            continue
        dist = euclidean([target['coor1'], target['coor2']],
                         [other['coor1'], other['coor2']])
        distances.append({'experiment_id': exp_id_other, 'distance': dist})

    distances = sorted(distances, key=lambda x: x['distance'])[:top_n]

    # Convert to similarity percentage (inverse normalised distance)
    max_dist = max(d['distance'] for d in distances) if distances else 1
    results = []
    for d in distances:
        match_row = enriched_df[enriched_df['experiment_id'] == d['experiment_id']]
        if match_row.empty:
            continue
        similarity = round((1 - d['distance'] / (max_dist + 1e-6)) * 100)
        results.append({
            'wine': match_row['wine'].values[0],
            'country': match_row['country'].values[0],
            'region': match_row['region'].values[0],
            'grape': match_row['grape'].values[0],
            'price': match_row['price'].values[0],
            'rating': match_row['rating'].values[0],
            'tier': TIER_NAMES[int(match_row['predicted_tier'].values[0])],
            'similarity': similarity
        })

    return results

def _fallback_similar(wine_row, enriched_df, top_n):
    """Fallback: same grape family and similar price tier"""
    primary_grape = wine_row['grape'].values[0].split(',')[0].strip()
    similar = enriched_df[
        enriched_df['grape'].str.contains(primary_grape, na=False) &
        (enriched_df['vintage_id'] != wine_row['vintage_id'].values[0])
    ].head(top_n)
    # format same as above
    return similar[['wine','country','region','grape','price','rating','predicted_tier']].to_dict('records')
```

```python
# Modify BiLSTM to return attention weights for interpretability
def get_attention_weights(model, review_text, vocab):
    model.eval()
    tokens = nltk.word_tokenize(review_text.lower())
    ids = torch.tensor([encode(review_text, vocab)], dtype=torch.long)

    with torch.no_grad():
        embedded = model.embedding(ids)
        output, _ = model.lstm(embedded)       # output: [1, seq_len, hidden*2]
        # Simple attention: dot product of each step with final hidden
        weights = torch.softmax(output.norm(dim=2), dim=1).squeeze()

    # Display top attended words
    word_weights = list(zip(tokens[:len(weights)], weights[:len(tokens)].tolist()))
    word_weights.sort(key=lambda x: x[1], reverse=True)
    print("Top attended words:")
    for word, weight in word_weights[:10]:
        bar = '█' * int(weight * 100)
        print(f"  {word:20s} {bar} {weight:.3f}")
```

## 6. Business Framing

### End user

A wine buyer at a restaurant group, hotel chain, or online retailer. Evaluates 50-200 new bottles per season. Current process: manually cross-referencing supplier sheets, Wine Advocate scores, and previous purchase history.

### What each model contributes

| Model | Question answered | Used how |
|---|---|---|
| CNN | What style and origin is this wine? | Classifies label → country + style |
| BiLSTM | How do people who drank this wine rate it? | Pre-processes all reviews → quality tier per vintage |

### Business decision logic

```python
def get_recommendation(country, tier):
    if tier == 2:  # Exceptional
        if country == 'France':
            return "Priority acquisition — classified estate profile. Feature on prestige list. Stock 6-12 bottles."
        else:
            return "High-value find — exceptional quality from non-French origin. Potential house special. Verify pricing."
    elif tier == 1:  # Premium
        return "Standard stock. Solid value at mid-range pricing. Suitable for main wine list. Review in 6 months."
    else:  # Entry
        return "Bulk purchase candidate or house wine. High volume, low margin. Check price per case."
```

### Cost of model errors

- False positive (Entry classified as Exceptional): buyer overpays, wastes prestige list space
- False negative (Exceptional classified as Entry): missed revenue opportunity
- False negative is more costly — erring toward caution is acceptable

### Ethical considerations

- Vivino review bias: more reviews from certain countries and price points
- Review language may correlate with reviewer demographics, not just wine quality
- Country classification from label aesthetics may reinforce design conventions rather than actual origin
- Price data from May 2023 — not current

## 7. Key Decisions Log

| Decision | Choice | Reason |
|---|---|---|
| Dataset | WineSensed (NeurIPS 2023) | Single dataset with image + text + napping, academically citable |
| CNN target | Country of origin (5 classes) | No wine type column in dataset; country has real visual signal |
| RNN target | Quality tier (3 classes) | Derived from Vivino rating; natural classification target |
| RNN at inference | Batch preprocessing | RNN runs offline, results stored; cleaner product, faster API |
| Image size | 224x224 | Required for ImageNet pretrained ResNet-18 |
| Max text length | 200 tokens | Covers 90%+ of reviews without truncation |
| Vocabulary | 20,000 words | Sufficient for wine vocabulary |
| Embeddings | GloVe-100d | Well-supported, good coverage for descriptive wine language |
| Similarity metric | Euclidean on napping coor1/coor2 | Direct from human taste experiment |
| Fallback similarity | Same primary grape + price tier | For wines outside 108-bottle napping subset |
| Deployment platform | Render | Free, native Python, no cold-start issues for demo |
